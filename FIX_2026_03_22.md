# Виправлення помилок від 22.03.2026

## Вирішені проблеми

### 1. ❌ Помилка з масивом images
**Помилка:** `malformed array literal: "["https://..."]"`

**Причина:** Неправильна конвертація JavaScript масиву в PostgreSQL array при використанні raw SQL.

**Рішення:** Виправлено функції `create` та `update` в `server/src/services/product.service.ts`:
```typescript
// Конвертація JS array в PostgreSQL array format
const imagesArray = Array.isArray(data.images) ? data.images : [];
const imagesPg = `{${imagesArray.map(img => `"${img.replace(/"/g, '\\"')}"`).join(',')}}`;
```

### 2. ❌ Нескінченна загрузка на сторінці товару
**Причина:** `productsApi.getBySlug()` повертає дані напряму, а не в `response.data`.

**Рішення:** Виправлено в `client/src/app/catalog/[id]/page.tsx`:
```typescript
// Було
const response = await productsApi.getBySlug(slug);
setProduct(response.data);

// Стало
const response = await productsApi.getBySlug(slug);
setProduct(response);
```

## ⚠️ ВАЖЛИВО: Міграція бази даних

**Перед розгортанням обов'язково виконайте міграцію БД на Railway!**

### Кроки:

1. **Відкрийте Railway Console** → ваша база PostgreSQL → **Console**

2. **Виконайте SQL:**
```sql
-- Додати поле slug
ALTER TABLE "Product" ADD COLUMN IF NOT EXISTS "slug" TEXT;

-- Заповнити slug для існуючих товарів
UPDATE "Product"
SET "slug" = LOWER(
    REGEXP_REPLACE(
        REGEXP_REPLACE(
            title,
            '[^a-zA-Z0-9а-яА-ЯїЇіІєЄґҐ-]',
            '-',
            'g'
        ),
        '-+',
        '-',
        'g'
    )
) || '-' || SUBSTRING(MD5(id || RANDOM()::TEXT) FROM 1 FOR 6)
WHERE slug IS NULL OR slug = '';

-- Індекс
CREATE INDEX IF NOT EXISTS "Product_slug_idx" ON "Product"("slug");

-- Унікальність
ALTER TABLE "Product" DROP CONSTRAINT IF EXISTS "Product_slug_key";
ALTER TABLE "Product" ADD CONSTRAINT "Product_slug_key" UNIQUE ("slug");

-- NOT NULL
ALTER TABLE "Product" ALTER COLUMN "slug" SET NOT NULL;
```

3. **Перевірте:**
```sql
SELECT id, title, slug FROM "Product" LIMIT 5;
```

## Якщо товар не відкривається

Можливі причини:

1. **Міграція не виконана** - поле `slug` відсутнє в БД
   - Рішення: виконайте міграцію (див. вище)

2. **Товар має старий ID в URL** - використовується `/catalog?id=...` замість `/catalog/slug`
   - Рішення: оновіть посилання в каталозі

3. **Slug не згенеровано** - товар створено до міграції
   - Рішення: виконайте UPDATE для існуючих товарів (див. міграцію вище)
